# Танковый бой 3D — деплой на бесплатные хостинги

Проект состоит из двух независимых частей:

- **`index.html`** — весь клиент игры (ангар, бой, графика, мультиплеер-клиент). Один файл, статика.
- **`server.js` + `package.json`** — relay-сервер мультиплеера на Node.js (WebSocket).

**GitHub Pages умеет отдавать только статику** (HTML/CSS/JS) и не может запускать
Node.js или PHP как постоянный процесс. Поэтому сервер разворачивается отдельно.

---

## Шаг 1. Деплой сервера (Render.com, бесплатно)

1. Зарегистрируйтесь на [render.com](https://render.com).
2. Загрузите папку с `server.js` и `package.json` в свой GitHub-репозиторий
   (можно в тот же репозиторий, что и клиент, просто в отдельную папку, например `server/`).
3. В Render: **New → Web Service** → подключаете репозиторий.
4. Настройки:
   - **Build Command:** `npm install`
   - **Start Command:** `node server.js`
   - **Instance Type:** Free
5. После деплоя Render даст адрес вида `https://your-app.onrender.com`.
   Для WebSocket используйте `wss://your-app.onrender.com` (тот же адрес, но протокол `wss://`).

**Важно про бесплатный тариф Render:** сервис "засыпает" после ~15 минут без
запросов и просыпается ~30-60 секунд при следующем подключении. Для игры с
друзьями это ок (первое подключение чуть медленнее), для постоянного продакшена
позже можно перейти на платный тариф или другой хостинг (Railway, Fly.io — тоже
есть бесплатные лимиты с похожими условиями).

---

## Шаг 2. Прописать адрес сервера в клиенте

Откройте `index.html`, найдите строку (поиском по `SERVER_URL`):

```javascript
let SERVER_URL = 'wss://YOUR-SERVER-ADDRESS.onrender.com';
```

Замените на реальный адрес вашего сервера с Шага 1.

---

## Шаг 3. Деплой клиента на GitHub Pages

1. Создайте репозиторий на GitHub (или используйте существующий).
2. Положите `index.html` в корень репозитория (или в папку `/docs`).
3. В настройках репозитория: **Settings → Pages → Source** — выберите ветку
   и папку (`/root` или `/docs`).
4. Через минуту-две игра будет доступна по адресу
   `https://ваш-логин.github.io/название-репозитория/`.

Готово — клиент на GitHub Pages, сервер на Render, всё бесплатно.

---

## Локальный тест перед деплоем

```bash
cd server/
npm install
node server.js
```

Сервер поднимется на `ws://localhost:8080`. Временно укажите в клиенте
`let SERVER_URL = 'ws://localhost:8080';`, откройте `index.html` в двух
вкладках браузера — они смогут найти друг друга по одинаковому коду комнаты.

---

## Как устроен мультиплеер (честно, без прикрас)

Это **relay-сервер**, а не полноценная авторитативная симуляция:

- Каждый клиент сам считает физику и попадания **своего** танка.
- Сервер только распределяет игроков по комнатам/командам и пересылает
  сообщения между клиентами одной комнаты.
- Каждый танк ~10 раз в секунду шлёт своё состояние (позиция, угол, HP) всем
  остальным в комнате — так синхронизируются позиции.
- При попадании стреляющий сам считает урон (бронепробитие, рикошет и т.д.)
  и отправляет результат цели — цель применяет урон к себе и разошлёт всем
  обновлённый HP следующим пакетом состояния.

**Ограничения этого подхода** (осознанный компромисс ради разумного объёма кода):
- Не защищено от читеров (клиент, которому доверяют, сам решает, попал он или нет).
  Для игры с друзьями это не проблема, для публичного сервиса — нужна честная
  серверная симуляция (совсем другой объём работы).
- Снаряды других игроков не рендерятся как летящие трассеры на вашем экране —
  вы увидите поворот их башни и почувствуете попадание (урон/вспышка), но не
  сам полёт снаряда. Это сделано, чтобы не дублировать физику снарядов на
  каждом клиенте и не бороться с рассинхроном baллистики.
- Таран между игроками физически расталкивает танки, но не наносит урон (чтобы
  не спорить, чей клиент "прав" в расчёте урона от столкновения).
- Комнаты и игроки хранятся в памяти сервера — при перезапуске сервера (в том
  числе автоматическом "просыпании" после сна на Render) все комнаты обнуляются.

Всё это — прочная основа, которую в будущем можно нарастить до полноценной
серверной симуляции, если понадобится честная защита от читеров.
